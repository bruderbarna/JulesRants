tűzfal:
	iptables (csomagszűrő):
		szűr: forrás és cél címe alapján, protokoll alapján (TCP/UDP), milyen portra, milyen portról
		iptables -L [OUTPUT] -n:
			szabályok (lánc, chain) kilistázása
			forward lánc: ha a gép átjáróként is üzemel, a továbbított forgalmat tudjuk filterezni
			ha nincs matching szabály, a policy lesz a meghatározó
		kimenő kapcsolat policijának dropra változtatása:
			iptables -P OUTPUT DROP
			iptables -P INPT ACCEPT stb.
		szabály hozzáadása:
			iptables -A (lánc végére)
			iptables -I (lánc elejére)
			példa:
		    iptables -A OUTPUT -d index.hu -j DROP
    törlés lehetőségek
      iptables -D OUTPUT -d index.hu -j DROP
      iptables -F OUTPUT
      iptables -F
    feladat: átjáró alhálózatához tartózó gépek közül csak az átjáró legyen elérhető
      route -n -nel megnézzük az átjárót és az alhálózat címét
      iptables -A OUTPUT -d 10.0.2.2 -j ACCCEPT
      iptables -A OUTPUT -d 10.0.2.0/24 -j DROP (alhálózatot dropoljuk)
      SORREND FONTOS
    feladat: index.hu-ról érkező csomagok ne tudják elérni a gépünket
      iptables -A INPUT -s index.hu -j DROP
    feladat:
      protokoll: iptables -A INPUT -p udp -j DROP

    DROP/REJECT különbség:
      ha drop eldobja, az csak eldobja
      ha reject eldobja, akkor visszajelez, hogy eldobta, és elvileg azonnal visszakapjuk a promptot
      reject - csengetnek és kiszólunk
      drop - csengetnek és hallgatunk

    iptables -A OUTPUT -d iatt.ttk.pte.hu -p tcp --dport 23 -j REJECT
     ha pl nem tudjuk a portot, használhatjuk helyette hogy pl. telnet

    feladat:
      www.pte.hu, csak a weboldalt lehessen elérni
      tagadás: iptables -A OUTPUT -p tcp ! --dport 80 -d www.pte.hu -j REJECT

    több port egy szabályban:
      iptables -A OUTPUT -d iatt.ttk.pte.hu -p tcp -m multiport --dport ssh,80,5432 -j ACCEPT

    szűrés mac címre:
      iptables -A INPUT -m mac --mac-source 01:32:42:EE:1B:12 -j DROP

    állapotkövetés:
      NEW,
      ESTABLISHED,
      RELATED (új szál nyitása),
      INVALID (érvénytelen),
      UNTRACKED (adott protokolloknál)

    feladat: minden established állapotút elfogadunk
      iptables -I INPUT -m state --state ESTABLISHED -j ACCEPT

    példa: állapotok vesszővel
      iptables -I INPUT -m state --state INVALID,UNTRACKED -j DROP

    szűrés interfacere:
      iptables -A INPUT -i eth0 -s 127.0.0.0/8 -j DROP
      iptables -A INPUT ! -i lo -s 127.0.0.0/8 -j DROP

    iptables -p icmp --help

    iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

    alhálózat gépeinek engedjük meg a pingelést:
      iptables -I INPUT -p icmp --icmp-type echo-request -s 10.0.2.0/24 -j ACCEPT
